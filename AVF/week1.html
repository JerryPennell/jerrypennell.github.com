Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

Twitter could be used for ‘live’ data resources for logins and payments for many various ways. One not so obvious way Twitter could be used for a login is for when you check-in to four square. When you log in or ‘check in’ it lets your friends know your participating. As for payments you might want to receive a coupon code prior to your next payment on an item.

Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

JSON-P would be used for script injection. JSON with padding allows you to get around security by using a HTTP GET. While JSON-P appears less secure if implemented correctly proves useful in proof of concept development too. JSON-P simply wraps the response in a function call named by the programmer. Cross origin resource sharing is the strength behind JSON-P.
